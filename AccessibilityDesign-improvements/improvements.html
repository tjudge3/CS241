Accessibility Changes and Improvements

Detailed below is information pertaining to the changes and improvments I have made from the original site to the revised site. In addition you will find data the shows the research performed, specifically in the form of screenshots.

Accessibility Inspector Results

Color Contrast Test

Semantic

I utilized h1, h2, and h3 as well as nav, section and caption in the website to create a more logical design to the website and improved accessibility for screen readers.

CSS rule selectors Changes

The font[size="7"], font[size="6"], font[size="5"] were changed to h1, h2, h3 respectively. div[class="nav"] was changed simply to nav

Image Changes

For the two images on the page, I added alt to explain what the images were. Additionally I added descriptions to the images in the paragraphs above each and included aria-labels.

Audio Player Changes

Previously the page simply stated "It looks like your browser doesn't support HTML5 audio players." as the error if the browser failed to load the player. I changed this to "It looks like your browser doesn't support HTML5 audio players. <a href="media/bear.mp3">Click here to access the file directly.</a>". I did the same for the audio narration I added to the page. In addition, I added an audio transcription and link to that audio transcription in the paragraph above the player <a href="transcription.html">This link contains a transcription of the audio</a>.

Form Changes

Previously in the from, "Your name:" and "Your comment:" did not have labels. I corrected that in the manner as described here : <label for="name">Your name: </label> and <label for="comment">Your comment: </label>

Comment Control Button Changes

I removed the show-hide selector from the css that defined the alignment and width, since it was not needed : .show-hide {text-align: center; width: 150px;}. Additionally I changed the div to a button: <div class="show-hide">Show comments </div> to <button class="show-hide">Show comments </button>

Table Changes

The first change I made to the table was to add a caption explaining the purpose of the table. From a code persective, I changed all the <td> to <th scope="col"> in the table head. Additionally I changed the first column (bear type) <td> to <th scope="row">.

Accessibility Change

Two of the accessibility changes that I added to the webpage were to narrate the article itself and add a transcription to the other audio file.

Other Changes

While not accessibility specific, I changed the pipe | to %7C <link href="https://fonts.googleapis.com/css?family=Open+Sans+Condensed:300%7CSonsie+One" rel="stylesheet" type="text/css"> so that the page would fully validate. I also changed the [if lt IE 9] <script src="http://html5shiv.googlecode.com/svn/trunk/html5.js"></script> [endif] to [if lt IE 9]> <script src="https://cdnjs.cloudflare.com/ajax/libs/html5shiv/3.7.3/html5shiv.js"></script> [endif] since the googlecode link 404's.
